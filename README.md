# Semiconductor Anomaly Detection & Process Control


ML-пайплайн для выявления брака (Pass/Fail) и поиска аномалий на основе данных с датчиков производственного оборудования. 

## Задача
Снижение процента брака на производстве путем предиктивной аналитики. 
Необходимо по показаниям сенсоров предсказать выход дефектной продукции, выявить ключевые факторы (датчики), влияющие на поломку, и настроить систему мониторинга для предотвращения будущих сбоев.

## Ключевые этапы пайплайна

1. **Предобработка без утечки данных (Data Leakage Prevention):**
   * Разделение на train/test со стратификацией.
   * Импьютация пропусков (`KNNImputer`) и масштабирование (`StandardScaler`) внутри кросс-валидации (Pipeline).

2. **Обучение и оптимизация:**
   * Алгоритм: `RandomForestClassifier`.
   * Подбор гиперпараметров через **Optuna** (максимизация метрики MCC на `StratifiedKFold`).
    
3. **Интерпретируемость (XAI):**
   * Использование **SHAP** (TreeExplainer) для понимания того, какие именно сенсоры вносят наибольший вклад в формирование брака.

4. **Статистический контроль процессов (SPC & MSPC):**
   * Проверка рядов на стационарность (ACF/PACF).
   * Построение контрольных карт Шухарта (I-MR) с использованием библиотеки [spc_lib](https://github.com/denccchick/spc_lib).
   * Многомерный анализ аномалий (MSPC) на основе **PCA**: вычисление статистик **Hotelling T²** и **SPE (Q-statistic)** для комплексного мониторинга процесса.

##  Результаты модели (Test Set)
Модель уверенно выявляет брак, несмотря на сильный дисбаланс, благодаря кастомному порогу классификации.

* **ROC-AUC:** `> 0.87`
* **MCC:** `> 0.55`
